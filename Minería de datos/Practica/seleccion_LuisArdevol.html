Eliminacin de variables con varianza baja

La clase VarianceThreshold es un enfoque bsico de seleccin de caractersticas que funciona eliminando todas las caractersticas cuya varianza no alcanza cierto umbral. Por defecto, elimina todas las caractersticas con varianza cero, es decir, aquellas caractersticas que tienen el mismo valor en todas las muestras.

Como ejemplo, supongamos que tenemos un conjunto de datos con caractersticas booleanas, y queremos eliminar todas las caractersticas que son cero o uno (apagado o encendido) en ms del 80% de las muestras. En este caso, las caractersticas booleanas son variables aleatorias de Bernoulli y su varianza est definida por:

$$Var(X)=p \cdot (1-p)$$

de forma que se pueda seleccionar utilizando el umbral $0.8 \cdot (1 - 0.8)$.

In[2]:
from sklearn.feature_selection import VarianceThreshold

X = pd.DataFrame([[0, 0, 1], [0, 1, 0], [1, 0, 0], [0, 1, 1], [0, 1, 0], [0, 1, 1]], columns=("A", "B", "C"))
sel = VarianceThreshold(threshold=(.8 * (1 - .8)))
X = pd.DataFrame(sel.fit_transform(X), columns=sel.get_feature_names_out())

for feature, variance in zip(sel.feature_names_in_, sel.variances_):
    print(feature, variance)

Seleccin de caractersticas univariante

La seleccin univariante de caractersticas funciona seleccionando las mejores caractersticas basadas en pruebas estadsticas univariantes. scikit-learn expone rutinas de seleccin de caractersticas como objetos que implementan el mtodo transform:

  • SelectKBest: elimina todas las caractersticas excepto aquellas con mayor puntuacin.
  • SelectPercentile: elimina todas las caractersticas excepto un porcentaje, especificado por el usuario, de las caractersticas con mayor puntuacin.
  • SelectFpr: elimina las caractersticas en base a tasa de falsos positivos.
  • SelectFdr: elimina las caractersticas en base a tasa de falsos descubrimientos.
  • SelectFwe: elimina las caractersticas en base a errores de la familia (se refiere a la probabilidad de cometer al menos un error de tipo I, es decir, rechazar una hiptesis nula verdadera).
  • GenericUnivariateSelect: permite realizar la seleccin univariante de caractersticas con una estrategia configurable.

Estos objetos toman como entrada una funcin de puntuacin que devuelve puntuaciones univariadas y/o p-valores:

  • Para regresin: r_regression, f_regression, mutual_info_regression
  • Para clasificacin: chi2, f_classif, mutual_info_classif

Los mtodos basados en el test F estiman el grado de dependencia lineal entre dos variables aleatorias. Por otro lado, los mtodos de informacin mutua pueden capturar cualquier tipo de dependencia estadstica, pero al ser no paramtricos, requieren ms muestras para una estimacin precisa.

Importancia de la seleccin de variables

En la prctica, para una correcta seleccin de variables sera necesario comprobar si el modelo entrenado se comporta correctamente con los datos de prueba. Para ello, vamos a modificar el ejemplo anterior aadiendo, adems, algo de ruido a los datos.

In[10]:
import numpy as np

from sklearn.model_selection import train_test_split

data, target = load_breast_cancer(return_X_y=True, as_frame=True)

# Generamos 20 columnas de ruido
noise = pd.DataFrame(np.random.default_rng().uniform(0, 0.1, size=(data.shape[0], 20)), columns=[f"noise_{i}" for i in range(20)])

combined = pd.concat([data, noise], axis=1)

# Partimos el dataset en entrenamiento y test para seleccionar las features y evaluar el clasificador
X_train, X_test, y_train, y_test = train_test_split(combined, target, stratify=target, random_state=42)

Que pasa con los resultados de accuracy en cada caso? Por que? Como vara el peso dado a las variable cuando no aplicamos seleccin de variables?

Antes de la seleccin de variables aadimos ruido a los datos. Esto sumado a las caractersticas del dataset original que eliminaba family-wise error con el estadstico F, dificulta que el modelo aprenda correctamente los patrones de las caractersticas realmente informativas y se fije ms en el ruido, dando un modelo ms complejo (an as, se obtiene un modelo con un 96.5% de accuracy usando todas las variables). Diferencias entre funciones de puntuacin

El siguiente ejemplo ilustra las diferencias entre el estadstico F y la informacin mutua.

Consideremos 3 caractersticas $x_1$, $x_2$, $x_3$ distribuidas uniformemente sobre [0, 1], donde el objetivo se define como:

$$y = x_1 + sin(6 * pi * x_2) + 0.1 * N(0, 1)$$

es decir, la tercera caracterstica $x_3$ es completamente irrelevante.

El cdigo que se muestra a continuacin traza la dependencia de y contra cada $x_i$ y los valores normalizados de las estadsticas de las pruebas F univariadas y la informacin mutua.

In[16]:
import matplotlib.pyplot as plt
import numpy as np

from sklearn.feature_selection import f_regression, mutual_info_regression

np.random.seed(0)
X = np.random.default_rng().random((1000, 3))
y = X[:, 0] + np.sin(6 * np.pi * X[:, 1]) + 0.1 * np.random.randn(1000)

f_test, _ = f_regression(X, y)
f_test /= np.max(f_test)

mi = mutual_info_regression(X, y)
mi /= np.max(mi)

plt.figure(figsize=(15, 5))
for i in range(3):
    plt.subplot(1, 3, i + 1)
    plt.scatter(X[:, i], y, edgecolor="black", s=20)
    plt.xlabel("$x_{}$".format(i + 1), fontsize=14)
    if i == 0:
        plt.ylabel("$y$", fontsize=14)
    plt.title("F-test={:.2f}, MI={:.2f}".format(f_test[i], mi[i]), fontsize=16)
plt.show()

Como la prueba F solo captura dependencia lineal, califica a $x_1$ como la caracterstica ms discriminativa. Por otro lado, la informacin mutua puede capturar cualquier tipo de dependencia entre variables y califica a $x_2$ como la caracterstica ms discriminativa, lo cual probablemente concuerda mejor con nuestra percepcin intuitiva para este ejemplo. Ambos mtodos marcan correctamente a $x_3$ como irrelevante.

Dado un estimador externo que asigna pesos a las caractersticas (por ejemplo, los coeficientes de un modelo lineal), el objetivo de la eliminacin recursiva de caractersticas (RFE, por sus siglas en ingls) es seleccionar caractersticas considerando recursivamente conjuntos de caractersticas cada vez menores. Primero, el estimador se entrena con el conjunto inicial de caractersticas y se obtiene la importancia de cada caracterstica ya sea a travs de un atributo especfico (como coef_, feature_importances_) o un callable. A continuacin, las caractersticas menos importantes se eliminan del conjunto de caractersticas y el proceso se repite recursivamente hasta que se alcanza finalmente el nmero deseado de caractersticas a seleccionar.

El problema de la eliminacin recursiva de caractersticas es que debemos determinar a priori el nmero de features que queremos conservar. Para solventar esta debilidad podemos emplear la variante RFECV, que realiza RFE en un bucle de validacin cruzada para encontrar el nmero ptimo de caractersticas. En ms detalle, el nmero de caractersticas seleccionadas se ajusta automticamente al ajustar un selector RFE en los diferentes splits de validacin cruzada (proporcionados por el parmetro cv). El rendimiento del selector RFE se evala utilizando un scorer para diferentes nmeros de caractersticas seleccionadas y se agregan juntos. Finalmente, las puntuaciones se promedian a travs de los pliegues y el nmero de caractersticas seleccionadas se establece en el nmero de caractersticas que maximiza la puntuacin de validacin cruzada.

Mtodos empotrados

Los mtodos empotrados o embebidos (embedded en ingls) son tcnicas de seleccin de variables que integran el proceso de seleccin directamente en el algoritmo de modelado. A diferencia de los mtodos basados en filtros, que seleccionan variables antes de construir el modelo, o de los mtodos basados en wrappers, que iteran sobre subconjuntos de variables, los mtodos embebidos realizan la seleccin de variables durante el proceso de entrenamento del modelo. Esto hace que sean ms eficientes y, en muchos casos, ms precisos.

Algunos ejemplos son:

  • La regresin Lasso, que aade una penalizacin L1 a la funcin de coste del modelo, lo que hace que los coeficientes de algunas variables se reduzan a cero, siendo eliminadas automaticamente.

  • La regresin Ridge, que aplica una regularizacin L2 y no realiza una seleccin de variables estricta pero penaliza los coeficientes grandes, lo que puede resultar til a la hora de reducir el impacto de las variables menos importantes.

  • Los algoritmos basados en rboles de decisin (incluidos los ensembles de estos), que miden la importancia de las variables en funcin de su contribucin a la reducin de la impureza al construir el rbol. Como consecuencia, proporcionan una mtrica de importancia de las variables, lo que permite seleccionar las ms relevantes.

Un mtodo frecuentemente usado como primera aproximacin debido a su simplicidade y precisin es el Random Forest. Este mtodo de ensemble entrena un gran nmero de rboles de decisin utilizando distintos subconjuntos de variables y datos. A continuacin toma la frecuencia de las variables ms representativas en los rbores con mejor rendimiento para estimar la importancia de cada caracterstica.

Una diferencia clave entre los mtodos embebidos y el resto es que los primeros pueden tener en cuenta la interaccin entre caractersticas, ya que el modelo aprende la importancia de las caractersticas de forma global, considerando como interactan entre si.

A continuacin tenemos un ejemplo de seleccin de caractersticas usando Random Forest.
